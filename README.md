## context 사용 <br />
- context를 사용해서 props 전달을 최소화하고, 필요한 곳에서 필요한 context 사용하는 방식으로 구현
- props 전달이 거의 없는 대신 Provider가 중첩으로 선언되어 복잡해짐, Provider 하위 컴포넌트 전체 리랜더링

## recoil 사용 <br />
- context api 대신 recoil을 사용하면 Provider depth가 줄어들고, 변경된 컴포넌트만 리렌더링
